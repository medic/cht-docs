## Highlights

### Collapsable contact summaries

Contact-summary cards are always fully expanded which, in cases like stock monitoring and immunizations, greatly  increases the amount of vertical scrolling needed to see the content below the card. With tihs feature, we now allow users to collapse and expand cards by tapping the header. Additionally, app developers can declare a property so cards can start collapsed or expanded as they desire. The the [issue](https://github.com/medic/cht-core/issues/10766) and the [docs](https://docs.communityhealthtoolkit.org/building/contact-summary/contact-summary-templated/#condition-cards) for more details.

### Custom UI Extensions

For many years  the community has been asking for ways to customize the UI of the CHT.  Whether was the desire for [configurable pages](https://forum.communityhealthtoolkit.org/t/the-ui-evolution-of-the-cht/3515/8) or a hope to [re-order the tabs](https://forum.communityhealthtoolkit.org/t/is-there-a-way-of-changing-the-order-of-the-tabs-of-cht/2093), deep, fully customizable UI for the CHT has been on going request for many years.  In 5.2 this feature is finally being delivered with UI Extensions.  By leveraging [web components](https://developer.mozilla.org/en-US/docs/Web/API/Web_components) the CHT now enables app developers to re-order tabs and create custom UIs which can call CHT APIs to read/create/update data for contacts, reports, etc.  See the `[docs](/building/reference/ui-extensions)` and [ticket](https://github.com/medic/cht-core/issues/102) for more info.

### Search tasks

During the [Smart Task Prioritization squad](https://github.com/medic/cht-roadmap/issues/191) it came up the CHWs who have a lot of tasks would benefit from being to use a free text search to quickly find tasks on their device.  As the tasks are all local, the search is instant and should greatly ease the process of scrolling over many tasks.  Check out [the ticket](https://github.com/medic/cht-core/issues/10694) for more info.

### Easily find failed replications 

As the CHT has grown to support larger and larger deployments, we're now seeing instances with over 20,000 active users and national deployments with over 80,000 active users spread across dozens of CHT instances.  With instances this important and this heavily utilized, it's critical to be able to trouble shoot any issues quickly.  One issue that Medic has seen cause real problems is having a user configured to syncronzie more than 10,000 documents. Not only can this cause the affected 10,000+ document users to fail to replicate, it can cause a slow down of the show CHT server. This makes the server slow for _all_ users.

In the past we've [discussed](https://forum.communityhealthtoolkit.org/t/cpu-impacts-of-users-with-10k-documents-how-to-fix-them/5413) how to trouble shoot users with too many docs, but CHT 5.2 introduces new, easier-to-use API which will list users who have issues replicating. Now CHT administrators will much more quickly be able to fix or disable these users to ensure healthy synchronizing for all users. See issue [#11146](https://github.com/medic/cht-core/issues/11146) and [#10794](https://github.com/medic/cht-core/issues/10794) for more details.

### Upgrade to CouchDB 3.5.2

Partnering with powerful open source projects has many benefits.  Aside from being able to leverage a massive body of like CouchDB which is finely tuned to deftly handling two way synchronizing of JSON documents, it's also always being improved.  CHT 5.2 includes CouchDB 3.5.2 which is now up 50% faster at indexing (making upgrades go faster) and up to 17% faster at indexing large amounts of documents.
