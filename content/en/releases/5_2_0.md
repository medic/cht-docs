## Highlights

### Collapsable contact summaries

Previously, the content fields for [contact summary condition cards](/building/contact-summary/contact-summary-templated/#condition-cards) were always visible to the user. Now, the body of the contact summary cards can be collapsed or expanded by tapping their headers. This reduces vertical scrolling for content-heavy cards like stock monitoring and immunizations. App developers can control the default state by setting a property on the card.

### Tab customizations

While the CHT has let you [hide tabs](https://forum.communityhealthtoolkit.org/t/is-it-possible-to-hide-certain-tabs/601/2?u=mrjones),  it has been impossible to set a new default tab or change the order of the tabs.  CHT 5.2 fixes that and enables deployments to choose the order and choose which tab a CHW sees when they first open their app. See [the ticket](https://github.com/medic/cht-core/issues/10908) for more details.


### UI Extensions

For many years  the community has [had a desire](https://forum.communityhealthtoolkit.org/t/the-ui-evolution-of-the-cht/3515/8) for fully customizable UI for the CHT.  In 5.2 this feature is finally being delivered with UI Extensions.  By leveraging [web components](https://developer.mozilla.org/en-US/docs/Web/API/Web_components) the CHT now enables app developers create custom UIs which can call CHT APIs to read/create/update data for contacts, reports, etc.  See the `[docs](/building/reference/ui-extensions)` and [ticket](https://github.com/medic/cht-core/issues/102) for more info.

### Search tasks

During the [Smart Task Prioritization squad](https://github.com/medic/cht-roadmap/issues/191) it came up the CHWs who have a lot of tasks would benefit from being to use a free text search to quickly find tasks on their device.  As the tasks are all local, the search is instant and should greatly ease the process of scrolling over many tasks.  Check out [the ticket](https://github.com/medic/cht-core/issues/10694) for more info.

### Identify users with replication failures

Large-scale CHT deployments are serving ever increasing numbers of users. With 20,000+ active users on some instances and 80,000+ users across multiple instances in national deployments, monitoring system health is critical.

Users configured to sync more than 10,000 documents can experience replication failures that impact both their own device performance _and_ the overall server response time for all users. Previously, identifying these problematic users required manual investigation and API queries. 

CHT 5.2 introduces a new API endpoint ([#11146](https://github.com/medic/cht-core/issues/11146) [#10794](https://github.com/medic/cht-core/issues/10794)) that quickly lists users who haven't successfully replicated within a configurable time interval. Administrators can now proactively identify and address users with excessive document loads, ensuring healthy synchronization for the entire deployment.

See also: [Monitoring user replication status](#monitoring-user-replication-status)

### Upgrade to CouchDB 3.5.2

The new CHT version includes CouchDB `3.5.2`, which brings a ~60% performance improvement for Nouveau indexing - the technology which powers the CHT replication algorithm and online full-text searches. This upgrade also [fixes an issue](https://github.com/medic/cht-core/issues/11153) causing search timeouts. See [the ticket](https://github.com/medic/cht-core/issues/11080) for more details.
